Here is the prompt and exact code to fix the iOS Status Bar issue.
This solution addresses the "White Flash" by priming the browser canvas with the correct dark brown color before React loads, and ensures the React layout doesn't immediately paint over it with cream/white on the landing page.
1. CSS Fix: In src/index.css, change the html, body background-color from #F2F2EC to #1a1610. This sets the underlying browser canvas to Dark Brown so it matches the Hero image during the initial load.
2. HTML Fix: In index.html, update the <meta name="theme-color"> for light mode and the inline script logic to use #1a1610 instead of #F2F2EC as the default.
3. Manifest Fix: In public/manifest.webmanifest, change background_color and theme_color to #1a1610.
4. Layout Fix: In src/App.tsx, update the Layout component to check if the current path is the landing page (/). If it is, set the wrapper background to #1a1610 and remove the top padding from <main> so the Hero image can extend behind the status bar."

2. index.html
HTML



<meta name="theme-color" content="#1a1610" media="(prefers-color-scheme: light)" />
<meta name="theme-color" content="#0f120a" media="(prefers-color-scheme: dark)" />

<script>
  (function() {
    var themeMode = localStorage.getItem('themeMode');
    var prefersDark = window.matchMedia && window.matchMedia('(prefers-color-scheme: dark)').matches;
    var isDark = themeMode === 'dark' || (themeMode === 'system' && prefersDark) || (!themeMode && prefersDark);
    
    // CHANGE: Default light color to #1a1610 (Dark Brown)
    var color = isDark ? '#0f120a' : '#1a1610';
    
    document.querySelector('meta[name="theme-color"]').setAttribute('content', color);
    if (isDark) document.documentElement.classList.add('dark');
  })();
</script>
