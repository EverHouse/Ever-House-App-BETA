### **1. üö® Critical: Admin Dashboard Will Crash**

Just like the Member Dashboard, the **Simulator Admin** (`src/pages/Admin/AdminDashboard.tsx`) uses `Promise.all` to fetch Requests, Pending Bookings, and Bays.

* **The Bug:** If *any* of those endpoints fail (e.g., the Bay service is down), the entire Admin Panel for simulators will crash or hang indefinitely.
* **The Fix:** Switch to `Promise.allSettled` to ensure staff can still manage requests even if one API is flaky.

### **2. üï∏Ô∏è SEO & Accessibility: "Buttons" vs "Links"**

In `Landing.tsx` and `Membership.tsx`, you are using `<button onClick={() => navigate('/...')}>`.

* **The Issue:** Search engines (Google) cannot follow these buttons. Users cannot "Right Click -> Open in New Tab". This hurts your site's SEO and usability.
* **The Fix:** Replace these navigation buttons with the `<Link>` component from `react-router-dom`.

### **3. ‚ö° Performance: Landing Page "Hero" Lag**

In `src/pages/Public/Landing.tsx`, the large hero image is set as a CSS `background-image` on a `div`.

* **The Issue:** Browsers load CSS background images *last*. This causes a "flash of invisible text" or a blank space while the page loads (poor LCP score).
* **The Fix:** Use a real `<img src="..." />` tag with `object-cover` positioned absolutely. Browsers prioritize `<img>` tags, making the page load visually faster.
