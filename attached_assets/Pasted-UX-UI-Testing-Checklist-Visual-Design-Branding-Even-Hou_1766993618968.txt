UX/UI Testing Checklist:

Visual Design & Branding
*  Even House green (#293515) appears consistently across the app
*  Lavender (#CCB8E4) accents appear where expected
*  Logos display correctly (dark/light versions switch with theme)
*  Playfair Display font shows on hero headlines
*  Inter font shows on body text and UI elements

Theme & Dark Mode
*  Light mode looks correct throughout
*  Dark mode looks correct throughout
*  System theme auto-switches correctly
*  Theme persists after closing/reopening app
*  Text is readable in both modes (WCAG AA contrast)

Glassmorphism & Effects
*  Glass blur effects appear on cards and modals
*  Buttons have proper hover/tap animations
*  Cards have subtle shadow and border effects
*  No harsh edges on glass elements

Animations & Motion
*  Hero parallax scrolling is smooth
*  Page transitions feel fluid (no jarring jumps)
*  Loading screen minimizes into status bar smoothly
*  Pull-to-refresh animation shows mascot correctly
*  Bottom nav hides/shows smoothly when scrolling
*  Staggered list animations work on content load
*  Booking confirmation sound plays

Navigation & Layout
*  Header shows mascot (hamburger), title, bell, avatar
*  Bottom nav icons are clear and tappable
*  Active nav item is clearly highlighted
*  Side menu opens/closes smoothly
*  Mascot in menu navigates home correctly
*  Back buttons work as expected

Mobile/PWA Experience
*  Safe area spacing correct on iPhone (notch, home bar)
*  Status bar shows brand green on landing page
*  No content hidden behind notch or home indicator
*  Haptic feedback triggers on button taps
*  Scrolling feels native and smooth
*  No horizontal overflow/scrolling
*  Modals respect safe areas and scroll properly

Responsive Design
*  iPhone - Layout fits, text readable, buttons tappable
*  iPad - Content uses space well, not stretched
*  Desktop - Centered layout, good use of whitespace

Loading States
*  Skeleton loaders appear while content loads
*  Loading spinners show during actions
*  No blank screens while waiting for data
*  Error states display clearly if something fails

Forms & Inputs
*  Input fields have clear labels and borders
*  Focus states are visible (accessibility)
*  Error messages appear inline and are clear
*  Success messages (toasts) appear and dismiss
*  Date/time pickers are easy to use

Accessibility
*  All buttons have sufficient tap target size (44x44px minimum)
*  Text has good contrast against backgrounds
*  Icons have labels or are self-explanatory
*  Modals can be dismissed by tapping backdrop

Typography & Readability
*  Text sizes are comfortable to read
*  Line spacing is adequate
*  Long text doesn't overflow containers
*  Event cards readable in light mode
*  Tier badges are legible
